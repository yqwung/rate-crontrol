3 码率控制模式

3 Rate Control

在介绍码率控制模式之前，首先整理一下工作中使用过的芯片平台编码器支持的码率控制模式，如下表：

平台 支持的码率控制模式 备注
Hisilicon CBR、VBR、AVBR、QVBR、CVBR、FIXQP、QPMAP
Novatek
Rockchip CBR、VBR
Horizon CBR、VBR、AVBR、FixQp、QPMAP
Axera CBR、VBR、AVBR、FixQp、QPMAP
SigmaStar CBR、VBR、AVBR、FIXQP
Ingenic CBR、VBR、CAPPED_VBR、CAPPED_QUALITY、FIXQP CAPPED_VBR 基于目标 PSNR 值的变码率模式、 CAPPED_QUALITY 基于 CAPPED_VBR 增加了用于解决剧烈运动下码率过低的现象的变码率模式

从上表可以看到，各家芯片的编码器支持的码率控制模式并不相同，但差异不大，基本都支持 CBR 和 VBR 码率控制模式，部分平台额外再加一些自家优化的码率控制模式。总的来说，常见的码率控制模式有 CBR、VBR、ABR、FIXQP、QPMAP 等，如下表：

码率控制 全称 描述
CBR Constant Bitrate 固定码率
VBR Variable Bitrate 可变码率
ABR Average Bitrate 平均比特率（目标比特率）
CQP Constant QP 恒定 QP ，同 FIXQP
FIXQP Fixed QP 恒定 QP ，同 CQP
QPMAP QP Map 自定义码控策略
CRF Constant Rate Factor 恒定速率因子（恒定画质）

编码器内部支持三种级别的码率控制，分别为帧级别、CTU/MB 级别和 subCTU/subMB 级别。其中帧级别的控制主要会根据目标码率为每一帧图片产生一个 QP 值，从而保证码率恒定；CTU/MB 级别的控制则根据每一个 64x64 的 CTU 或 16x16 的 MB 的目标码率为每个 block 产生一个 QP 值，能够得到更好的码率控制，但是频繁的 QP 值调整会造成图像质量不稳定的问题；subCTU/subMB 级别的控制则为每一个 32x32 的 subCTU 或 8x8 的 subMB 产生一个 QP 值，其中复杂的块会得到较高的 QP 值，静态的块则会得到较低的 QP 值，因为相比于复杂的区域人眼对于静态的区域更敏感，复杂和静态区域的检测主要依赖于内部硬件模块，这个级别控制主要是为了提高主观图像质量同时保证码率恒定，该模式控制下 SSIM 得分较高，但是 PSNR 得分会降低。

不同的应用场景需要选用不同的码率控制模式，来满足需求，达到尽可能好的编码效果。
